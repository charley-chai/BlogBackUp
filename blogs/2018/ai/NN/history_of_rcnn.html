R-CNN简史

OCT 05, 2018

目标检测

什么是R-CNN呢？首先从计算机视觉（Computer Vision）说起。在计算机视觉领域中最常见的任务为：
  • 语义分割（semantic segmentation）
  • 目标检测（object detection）
  • 目标识别（object recognition）
  • 实例分割（instance segmentation）
而我们这里将要接触的R-CNN就是为了解决目标检测的问题。所谓目标检测就是：给定一张图像，找出出所有的目标并标注出相应的类别。 由于物体的尺寸有大有小，摆放的角度、姿态等多种多样不定，物体可以出现在图片的任何地方；此外，物体的种类不可谓不繁杂。所以，这个任务其实是相当复杂的。

$\triangle$ 目标检测的例子

目前的目标检测算法分成3类：
  • 传统的目标检测算法：Cascade + HOG/DPM + Haar/SVM
  • 候选区域/窗 + 深度学习分类：通过提取候选区域，并对相应区域进行以深度学习方法为主的分类的方案，如：
    • R-CNN（Selective Search + CNN + SVM）
    • SPP-net（ROI Pooling）
    • Fast R-CNN（Selective Search + CNN + ROI）
    • Faster R-CNN（RPN + CNN + ROI）
      • ......
  • 基于深度学习的回归方法：YOLO、SSD、DenseBox 等

传统的目标检测算法

要完成目标检测，CNN已经可以完成图像识别的任务，而我们做一些额外的工作来完成定位任务即可。这里介绍两个思路：

#001

滑动窗口


使用不同尺度的滑动窗口选定图像的某一区域为候选区域。




#002

提取特征


从对应的候选区域提取如Harr HOG LBP LTP等一类或者多类特征。

#003

分类判断


使用Adaboost SVM 等分类算法对对应的候选区域进行分类，判断是否属于待检测的目标。




候选区域/窗 + 深度学习分类的目标检测算法

下面按照时间顺序简单介绍一下在目标检测领域表现很好的R-CNN的发展过程。

2013

R-CNN横空出世


Ross Girshick设计的R-CNN（Region CNN）是利用深度学习进行目标检测的开山之作。经典的目标检测算法使用滑动窗法依次判断所有可能的区域，而R-CNN则预先提取一系列较可能是物体的候选区域，之后仅在这些候选区域上提取特征，进行判断，最终得到的结果是：图像中每个物体的Bounding Box+标签。

但是，RCNN有严重的速度瓶颈，因为每次都要对同一图像中有巨大重叠的候选区域分别提取特征，在特征提取过程中有极大的重复计算。
而且在特征提取过程中，候选区域的大小并不一致，在RCNN中将这些区域拉伸为统一大小，这导致候选区域失真。独立的分类器和回归器需要大量特征作为训练样本。
CNN提取图像特征、分类器预测类别、回归模型tighten bounding box的边界都是分别训练的，这也导致流程不易训练。

2015

Fast R-CNN


针对R-CNN的上述缺点，Ross Girshick又提出了改进版本——Fast R-CNN。他发现，每张图像的proposed regions总是朴素重叠的，于是将整张图像归一化后直接送入CNN计算特征图。RoIPool分享整幅图像的CNN前向计算给不同的regions。将CNN、分类和边界线性回归的训练融合到一个单独的网络中实现，不再需要额外存储。

2015

Faster R-CNN


经过上述的改进，Fast R-CNN仍然存在一个瓶颈——region proposer。微软研究人员Shaoqing Ren（中科大校友😉）等人将Fast R-CNN进一步改进，提出一种Faster R-CNN的方法，使得region proposal非常高效。在结构上，Faster R-CNNN将特征抽取(feature extraction)，proposal提取，bounding box regression(rect refine)，classification都整合在了一个网络中，解决了Fast R-CNN的瓶颈问题，使得综合性能有较大提高，在检测速度方面尤为明显。

Faster RCNN首先使用一组基础的conv+relu+pooling层提取image的feature maps；RPN网络用于生成region proposals；Roi Pooling层收集输入的feature maps和proposals，综合这些信息后提取proposal feature maps，送入后续全连接层判定目标类别；Classification层利用proposal feature maps计算proposal的类别，同时再次bounding box regression获得检测框最终的精确位置。



论文链接：Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks


2017

Mask R-CNN


至此，我们可以快速定位图像中不同目标的bounding boxes了，那么如何达到如图所示的像素级的目标检测呢？



Facebook Research的Kaiming He团队提出了Mask-RCNN。Maks R-CNN在Faster R-CNN的基础上增加一个分支，用于输出一个二值掩膜（Binary Mask），判断给定像素是否属于物体。

其结构如图所示，在原来Faster R-CNN的基础上增加了Mask（如图其中黑色部分为原来的Faster-RCNN，红色部分为在Faster网络上的修改）。



Mask R-CNN采用RoIAlign的方法对RoIPool进行调整使其更精确对齐。生成这些二值掩膜（Binary Mask）之后，Mask R-CNN结合Faster R-CNN中生成的分类和bounding box，生成精确的分割结果。
